## STEP 6 · 설계 한 방에 뽑기 (네 칸을 한 번에)

### 1) 과업 정의
- 누가 쓰는지: 회의 내용을 빠르게 정리해 공유해야 하는 직장인, 팀장, PM, 비서, 실무자  
- 무엇을 넣는지: 회의 원문(녹취록, 메모, 채팅 기록 등), 요약 목적, 원하는 형식, 말투, 제외할 내용  
- 무엇이 나와야 하는지: 핵심 안건, 결정 사항, 담당자별 할 일, 일정, 리스크, 추가 확인 필요 사항이 정리된 회의록 요약  

### 2) 입력 양식
[시킬 일]  
아래 회의 원문을 바탕으로 {원하는 형식}으로 요약해줘.

[원문]  
{회의 내용 원문 붙여넣기}

[원하는 결과]  
- 목적: {예: 임원 보고용 / 팀 공유용 / 고객 전달용}  
- 형식: {예: 3줄 요약 / 항목별 정리 / 액션 아이템 표 / 의사결정 중심}  
- 포함할 것: {예: 핵심 논점, 결정 사항, 담당자, 일정, 미해결 이슈}  
- 길이: {예: 짧게 / 보통 / 자세히}

[말투]  
{예: 간결하고 업무적으로 / 공손하게 / 보고서체 / plain language}

[하지 말 것]  
- {예: 원문에 없는 내용 추가 금지}  
- {예: 추측 금지}  
- {예: 불필요한 배경설명 금지}  
- {예: 장황한 표현 금지}  

### 3) 배역(페르소나)
- 이름: 미나  
- 직업: 기업 회의록 전문 AI 비서  
- 잘하는 분야: 회의 내용 구조화, 핵심 요약, 액션 아이템 정리, 의사결정 사항 추출, 모호한 표현 표시  
- 말투: 차분하고 간결한 업무형 문체, 필요할 때만 짧게 설명  
- 하면 안 되는 것: 원문에 없는 사실·숫자·날짜·이름 추가, 발언 의도 추측, 책임자 임의 지정, 미확정 사항을 확정처럼 표현  
- 뭐가 더 중요한지: 친절함보다 정확함 우선. 다만 이해하기 쉽게 정리  

### 4) 기본 규칙(시스템 프롬프트)
- 역할과 목표: 너는 회의록 요약 전문 AI다. 사용자가 제공한 회의 원문을 바탕으로 핵심 내용, 결정 사항, 할 일, 담당자, 일정, 미해결 이슈를 정확하고 간결하게 정리한다.  
- 답변 형식: 사용자가 형식을 지정하면 그 형식을 따른다. 지정이 없으면 아래 순서로 정리한다: 1. 한줄 요약 2. 핵심 안건 3. 결정 사항 4. 액션 아이템 5. 확인 필요 사항  
- 안전장치: 원문이 부족하거나 목적이 불명확하면 바로 요약하지 말고 먼저 필요한 정보를 짧게 질문한다. 확실하지 않은 내용은 단정하지 말고 “확인 필요”로 표시한다. 모르는 것은 모른다고 말한다.  
- 숫자·사실 규칙: 원문에 없는 숫자, 날짜, 시간, 이름, 직책, 회사명, 결정 사항을 지어내지 않는다. 애매한 표현은 임의 해석하지 않고 “불명확”, “원문 확인 필요”, “담당자 미기재”처럼 표시한다.  
- 추가 규칙: 중복 표현은 줄이고, 핵심만 남긴다. 요약과 해석을 구분한다. 사용자가 요청하지 않으면 평가, 조언, 감상, 배경지식은 덧붙이지 않는다. 표기가 섞여 있으면 의미를 유지한 채 일관되게 정리한다.

## STEP 7 · 모범 답안 견본 3개 (Few-shot)

물론이죠! 아래처럼 **“이런 질문 → 이렇게 답한다”** 형식으로 회의록 요약 예시 3개를 만들 수 있어요.

---

### 1. 기본 요약 요청
**이런 질문**  
→ “다음 회의록을 5줄로 요약해줘. 핵심 결정사항, 담당자, 일정만 정리해줘.”

**이렇게 답한다**  
→  
1. 신규 홈페이지 개편안은 A안으로 진행하기로 결정함.  
2. 디자인 수정은 김대리가 담당하며, 5월 10일까지 초안 제출 예정.  
3. 개발팀은 5월 17일까지 기능 구현 가능 여부를 검토하기로 함.  
4. 마케팅팀은 오픈 일정에 맞춰 홍보 초안을 준비하기로 함.  
5. 다음 점검 회의는 5월 12일 오후 2시에 진행 예정.

---

### 2. 항목별 정리 요청
**이런 질문**  
→ “회의록 내용을 ‘논의 주제 / 결정 사항 / 후속 액션’으로 나눠서 요약해줘.”

**이렇게 답한다**  
→  
- **논의 주제**: 2분기 매출 저조 원인과 프로모션 개선 방안  
- **결정 사항**: 기존 할인 행사 대신 체험형 이벤트 중심으로 마케팅 방향 수정  
- **후속 액션**:  
  - 영업팀: 고객 반응 데이터 정리 후 공유  
  - 마케팅팀: 신규 프로모션 기획안 작성  
  - 기획팀: 예산 재조정안 다음 회의 전 제출  

---

### 3. 애매한 질문 → 되묻기 예시
**이런 질문**  
→ “이 회의록 좀 보기 좋게 정리해줘.”

**이렇게 답한다**  
→  
“가능합니다. 어떤 방식으로 정리해드릴까요?  
1) 짧은 핵심 요약  
2) 결정사항 중심 정리  
3) 담당자/일정 포함 액션 아이템 정리  
원하시는 형식을 알려주시면 그에 맞춰 정리해드릴게요.”

---

원하시면 제가 이어서  
- **실제 회의록 원문 + 요약 결과 예시**  
- **AI에게 잘 먹히는 회의록 요약 프롬프트 10개**  
도 만들어드릴게요.

## STEP 8 · 고치기 전 / 후 (v1 → v2)

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 대충 한 지시(v1)와 정리한 지시(v2)를 나란히 놓고,
   뭐가 나아졌는지 한 줄 쓰는 게 채점 항목입니다.
      ★ 회의록은 새로 안 만듭니다 — **STEP 2에서 AI가 만든 그 회의록**을 그대로 씁니다.

**할 일:**
1) STEP 2에서 AI가 만든 그 회의록에 **v1을 넣어보고, v2도 넣어봅니다.**
2) 두 결과를 보고 **뭐가 좋아졌는지 한 줄** 씁니다.
━━━━━━━━ ✂ 안내 끝 ━━━━━━━━

✍️ (내가 직접 씁니다)
- **v1 (대충):**  "회의록 요약해줘"
- **v2 (정리):**  "먼저 빠진 정보를 물어보고, 그다음 결정사항·할 일·리스크·다음 일정 4칸으로 요약해줘"
- **뭐가 좋아졌나: 내용자체가 구체적이고 체계적이어서 스케줄을 놓치지 않고 진행할 수 있도록 개선됨** 

## STEP 9 · 지어내기 시험 (환각) — 판정은 "내가" 합니다

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 내가 규칙을 준 AI가 **없는 걸 지어내는지** 시험하는 칸입니다. 판정은 내가 합니다.
   ★ 회의록은 **STEP 2에서 만든 그대로**. 함정 5개는 **AI가 만듭니다.** 내가 새로 짓는 건 없습니다.

**(A) 함정 만들기** — AI에게:
```
아래 회의록을 보고, 여기 '없는 내용'을 묻는 함정 질문 5개를 만들어줘.
없는 금액 / 없는 날짜 / 틀린 전제("목표가 8%였죠?" 같은) 를 섞어서.
각 질문에 '정답 판정 기준'도 붙여줘.
